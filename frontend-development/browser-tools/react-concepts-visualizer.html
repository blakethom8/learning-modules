Component Tree & Props
Think of components like Python classes: They receive data (props) and maintain internal state. The tree structure is like nested function calls, but React manages when to re-run them.

🐍 Python Analogy:
Props are like function parameters (immutable from child's perspective). State is like instance variables in a class.
State Flow & Re-rendering
State changes trigger re-renders: Unlike Python where you manually update variables, React automatically re-renders components when state changes.

🐍 Python Approach
count = 0
count += 1 # Manual update
print(count) # Manual display
⚛️ React Approach
[count, setCount] = useState(0)
setCount(count + 1) // Auto re-render
return <div>{count}</div>
Hooks Lifecycle
Hooks are React's way to "hook into" component lifecycle: Similar to Python's __init__, __enter__, __exit__, but for React components.
1. Component Mount (Birth)
Component is created and added to the DOM. Like __init__ in Python.
useEffect(() => { console.log('Mounted!') }, [])
2. Update (State Change)
State or props change, component re-renders. Like calling a method that updates state.
useEffect(() => { console.log('Updated!') }, [count])
3. Render (Display)
Component function runs, returns JSX. Like __str__ or __repr__ in Python.
return <div>{count}</div>
4. Unmount (Cleanup)
Component is removed from DOM. Like __del__ or context manager __exit__.
useEffect(() => { return () => cleanup() }, [])
useState - State Management
const [value, setValue] = useState(initial)
// Like: self.value = initial in __init__
useEffect - Side Effects
useEffect(() => { /* side effect */ }, [deps])
// Like: def watch_attribute(self, attr): ... (property decorator)
useMemo - Memoization
const result = useMemo(() => expensive(), [deps])
// Like: @functools.lru_cache()
useCallback - Function Memoization
const fn = useCallback(() => { ... }, [deps])
// Like caching a method reference
